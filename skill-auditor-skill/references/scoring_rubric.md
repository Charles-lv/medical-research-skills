# Skill 评测评分量表与红线标准

## 第一步：基础版一票否决 (Stability & Security)
任何一项 FAIL 则直接拒绝。

| 维度 | 判定条件 | 结果 |
| :--- | :--- | :--- |
| **T1. 运行稳定性** | 1. 失败率 > 20%；2. 存在随机 Crash 或死循环；3. 依赖冲突需人工干预。 | PASS / FAIL |
| **T2. 结构一致性** | 1. 缺失 API 字段或非合法 JSON；2. 返回类型不一致；3. 字段名不固定。 | PASS / FAIL |
| **T3. 结果确定性** | 1. 低温下输出差异巨大；2. 缺乏 Seed 管理；3. 关键数值随机波动。 | PASS / FAIL |
| **T4. 系统安全性** | 1. 执行原始字符串代码；2. 无输入过滤；3. 存在 Prompt Injection 风险。 | PASS / FAIL |

## 第二步：基础必评项 (基础分：40分)

| 维度 | 评估重点 | 满分 |
| :--- | :--- | :--- |
| **1. 场景精度与触发** | 触发词自然、覆盖广；描述具体（如“优化 Python 异步函数”而非“写代码”）。 | 10 |
| **2. 架构分层与精简** | 指令干练，无废话；复杂信息通过 References 分级加载。 | 10 |
| **3. 动作闭环与流转** | 提供直接可用的代码/路径；包含“执行-验证-反馈”闭环。 | 10 |
| **4. 覆盖完备与差异化** | 定义明确的“是什么”和“何时用”；避免触发词重叠冲突。 | 10 |

## 第三步：分类
1. **科研设计 / 数据分析类** (实验设计、R/Python 分析、统计方案)
2. **科研文本理解 / 学术写作类** (综述生成、Cover Letter、方法改写)
3. **其他** (通用非科研类)

## 第四步：专项评判 (专项分：60分)
*注：仅针对科研类 Skill。*

| 维度 | 评分标准 | 满分 |
| :--- | :--- | :--- |
| **1. 术语准确性** | 完美使用医学标准术语 (MeSH, MedDRA)；精准区分关联与因果。 | 20 |
| **2. 逻辑严谨性** | 论证逻辑严密，无因果断裂，有生物医学逻辑支撑。 | 15 |
| **3. 证据等级与解读** | 准确识别证据等级 (RCT vs. 观察性)；实验设计描述专业。 | 15 |
| **4. 学术风格匹配度** | 纯正 SCI 论文风格；缩写规范；无营销口吻。 | 10 |

## 第五步：科研版一票否决 (Medical Integrity)
*注：仅针对科研类 Skill。任何一项 FAIL 则直接拒绝。*

| 维度 | 判定条件 | 结果 |
| :--- | :--- | :--- |
| **M1. 学术诚信** | 1. 编造 DOI/PMID；2. 捏造临床数据或实验结果。 | PASS / FAIL |
| **M2. 执业边界** | 1. 直接诊断建议；2. 缺少免责提示；3. 推荐未获批方案；4. 泄露隐私。 | PASS / FAIL |
| **M3. 方法学底线** | 1. 严重方法学错误（如因果倒置）；2. 忽略伦理合规需求。 | PASS / FAIL |
| **M4. 代码可用性** | 1. 生成不可运行代码（语法错误、逻辑死循环、缺失核心依赖）。 | PASS / FAIL |

## 第六步：总分计算与发布建议

### 计算公式
- **科研类**: `总分 = 基础分 + 专项分`
- **其他类**: `总分 = 基础分 * 2.5` (换算为100分制)

### 发布建议等级

| 分数 | 等级 | 发布建议 |
| :--- | :--- | :--- |
| 85–100 | Production Ready | 可对外发布 |
| 75–84 | Limited Release | 限流测试 |
| 60–74 | Beta Only | 仅内部/灰度 |
| <60 | Reject | 不上线 |
