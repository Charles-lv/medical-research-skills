## Mode B — Open Science Best Practices

### Pre-registration
- [ ] Pre-registration completed before data collection/analysis
- [ ] Appropriate platform selected (OSF / AsPredicted / ClinicalTrials.gov / PROSPERO / INPLASY)
- [ ] Confirmatory and exploratory analyses clearly distinguished
- [ ] Deviations from pre-registration documented and justified

### FAIR Data
- [ ] Data deposited in trusted repository with persistent identifier (DOI)
- [ ] Data dictionary / codebook documenting every variable
- [ ] Standard formats used (CSV, JSON, HDF5)
- [ ] Controlled vocabularies applied (GO, MeSH, EFO)
- [ ] License specified for data (CC-BY, CC0) and code (MIT, Apache)
- [ ] FAIR principles self-assessment completed

### Code and Computational Environment
- [ ] Analysis code shared in public repository (GitHub/GitLab)
- [ ] Zenodo DOI minted for code release
- [ ] All dependencies documented (requirements.txt / renv.lock / environment.yml)
- [ ] Dockerfile or Binder configuration provided
- [ ] README with execution instructions included
- [ ] Code tested on clean environment

### Replication Study Design
- [ ] Replication type defined (direct / conceptual)
- [ ] Power analysis based on original effect size with safeguard adjustment
- [ ] Success criteria pre-specified (equivalence bounds / Bayes factor)
- [ ] All materials sufficient for independent replication

### Reporting Transparency
- [ ] Reporting guideline checklist completed (CONSORT / STROBE / ARRIVE / PRISMA / STARD / TRIPOD)
- [ ] All pre-specified analyses reported regardless of results
- [ ] Exploratory analyses clearly labeled
- [ ] Supplementary materials shared

### Open Science Practices
- [ ] Open science badge eligibility assessed (data / materials / pre-registration)
- [ ] Registered reports format considered
- [ ] Preprint posted (bioRxiv / medRxiv / arXiv / SSRN)
- [ ] Open access publication route selected
